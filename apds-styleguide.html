What's new

Recent additions and changes synced from Figma.

2026-05-18

Interactive — Filled red variant added

A new interactive/filled-red variant joins Filled, Outlined, Overlay, Text link, and Text link subtle. It includes background, icon, and label tables for a primary red CTA treatment (used for donation and urgency actions). The enabled/active/selected states all resolve to red.600 (#da1026); hover-bg uses the new primitive red.400A (#ff1818) to brighten on hover. See Colors → Interactive → Filled red.

2026-05-07

Line-height tokens added

Five new line-height tokens are now part of the typography utility set: tight (1.1), heading-120 (1.2), heading-130 (1.3), body (1.5), and relaxed (1.8). Documented under Font weight & stretch → Line height.

2026-05-07

Spacing scale is now responsive

All spacing tokens (spacing-100 through spacing-900 and screen-margins) resolve to larger values at the LG and XL breakpoints. The Spacing section now shows values at every breakpoint instead of just SM.

2026-05-07

Interactive — Overlay color group added

A new interactive/overlay variant joins Filled, Outlined, Text link, and Text link subtle. It includes background, border, icon, and label tables for use on overlays sitting above media or imagery (e.g. play buttons, hero scrims). See Colors → Interactive → Overlay.

2026-05-07

Text-style token mappings audited

Every Typography row was cross-checked against the variables JSON. Corrections:

  • Utility/Description LG, MD, SM → fixed-lg, fixed-md, fixed-sm (non-responsive). Description LG also resizes from 20px to 18px.
  • Story links/List item LG, MD, SM → fixed-lg, fixed-md, fixed-sm (List item LG was incorrectly labelled x-base at 18px — x-base is 16px).
  • Utility nav links (Main, Trending, Section, Storyline) → nav-link, the dedicated fixed 14px token.
  • Utility/Timestamp, Metadata Reg, Metadata Bold → metadata, the dedicated fixed 12px token.

Text sizes (type scale)

Text sizes are responsive: smaller at SM and MD breakpoints, larger at LG and XL. Styles that use the x-scale (x-base, x1, x2, etc.) follow this two-tier scale. Styles in the Special group can have a different size at each of the four breakpoints (SM, MD, LG, XL).

X-scale (responsive)

Smaller at SM/MD, larger at LG/XL. Use var(--ap-text-size-x-base) etc.; in production these resolve per breakpoint via media/container queries.
